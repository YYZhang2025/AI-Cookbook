Tensor Parallelism

Apr 03, 20265 min read

  • LLM
  • Hardware
  • Parallelism

我们有一个Linear Layer:

Y=XW⊤+b

其中：

  • X∈RB×H
  • W∈RO×H
  • Y∈RB×O

如果H 和 W 很大，比如在大模型里上万甚至更大，这个 W 非常占显存。这时就可以把W 切开，放在多张GPU上。Tensor Parallelism有两种不同的切法：

  • Column Parallel： 把权重矩阵 W 按输出维度切开
  • Row Parallel: 把权重矩阵 W 按输入维度切开：

Column Parallel

Column Parallel 的核心思想是：

**把线性层的输出维度** O **切分到多张 GPU 上。**

也就是说，虽然完整权重是 W∈RO×H, 但我们不让每张 GPU 都保存完整的 W，而是把它按 输出行 切开。因为在 PyTorch 里 Linear 的权重形状通常写成 [out_features, in_features]，所以“按输出维切”对应到矩阵上，就是按第 0 维切分, 假设我们有 p z长GPUs，那么 W 可以写成：

W=​W1​W2​⋮Wp​​​,Wi​∈RpO​×H

对应的，如果我们有bias term，它也将对应切开：

b=​b1​b2​⋮bp​​​,bi​∈RpO​

然后每张 GPU 独立计算自己的局部输出：

Yi​=XWi⊤​+bi​

其中：

Yi​∈RB×pO​

最后我们只需要将所有GPU上的局部输出，沿着输出维度拼接起来即可：

Y=[Y1​,Y2​,…,Yp​] XW⊤=X[W1⊤​​W2⊤​​…​Wp⊤​​]=[XW1⊤​,XW2⊤​,…,XWp⊤​]

Backward

与Forward Pass类似，Backward Pass只需要负责自己需要的那一部分，但是对于输入维度，我们需要进行all-reduce。

  • dW 是局部可算的

  • db 是局部可算的

  • dX 需要 all-reduce

Row Parallel

Row Parallel 的核心思想是：

把线性层的输入维度 H 切分到多张 GPU 上。

W=[W1​,W2​,…,Wp​]

其中每一块都是：

Wi​∈RO×pH​

与Column Parallelism不同的是，输入 X 也必须对应切开：

X=[X1​,X2​,…,Xp​]

其中：

Xi​∈RB×pH​

我们看一下每张GPU拿到什么：

    • 自己那部分输入 X_i
    • 自己那部分权重 W_i

它可以独立计算局部贡献：

Zi​=Xi​Wi⊤​

需要注意的是，Zi​∈RB×O

也就是说，每张 GPU 算出来的形状已经是完整输出形状，但它只是完整输出的一部分“贡献”，不是最终结果。

最后，把所有 GPU 的局部结果相加：

Y=∑i=1p​Zi​+b

也就是说，最后是 all-reduce 的操作

FFN

我们可以将Column Parallelism与 Raw Parallelism 结合起来，举个例子，对于Transformer中的FFN Layer，它有：

Y=σ(XW1⊤​)W2⊤​

这里我们忽略bias的term

通常，我们会将第一层用于Column Parallelism, 第二层用于Row Parallelism，why？

xColumn Parallel​hlocal​GELU​hlocal​Row Parallel​y

这一层的特点是：输出维很大，通常是 hidden size 的 4 倍。于是每张卡只负责中间激活的一部分：而 FFN 中间维本来就很大，所以按这个维度切最合理。

而第二层的Row Parallelism，需要提前切好输入 X, 但是在第1层，我们已经完成了风格，因此减少了通信。

Attention

在 Attention 里，Tensor Parallelism 的思路和 MLP 很像：

  • 前面的投影层把大维度切开

  • 每张 GPU 只算自己那部分 heads

  • 最后的输出投影再把各张卡的结果汇总

最常见的写法就是：

  • q_proj / k_proj / v_proj 用 Column Parallel

  • o_proj 用 Row Parallel

因为 attention head 天然就是可分的。

比如总共有 32 个 heads，用 4 张 GPU，那么可以很自然地分成：

  • GPU0：8 个 heads

  • GPU1：8 个 heads

  • GPU2：8 个 heads

  • GPU3：8 个 heads

每张 GPU 只负责自己那部分 heads 的：

  • Q

  • K

  • V

  • attention score

  • attention output

也就是说，Attention 的并行粒度天然就是 按 head 切。

这也是 Attention 比很多别的结构更适合 TP 的原因。

(Shoeybi et al., 2020)
